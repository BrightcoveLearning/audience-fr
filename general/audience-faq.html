    FAQ Audience Brightcove

    Cette rubrique contient certaines des questions les plus fréquentes posées à propos de Brightcove Audience.

    Pourquoi le public et comment cela fonctionne

    Pourquoi dois-je suivre la vidéo dans ma solution d'automatisation marketing ?
    Les solutions d'automatisation marketing ne suivent pas la vidéo aujourd'hui. Si vous investissez dans la vidéo en tant que contenu puissant, vous devez vous assurer d'en suivre l'efficacité et d'automatiser les communications avec les leads/utilisateurs qui consultent votre contenu.
    Brightcove et le module Audience utilisent-ils des cookies ?
    Le module Audience crée et utilise un cookie tiers pour identifier les utilisateurs lorsqu'une intégration est en place. De cette façon, les spectateurs ne sont pas invités à remplir un formulaire de prospect une deuxième fois. Le module utilise également les API Marketo, HubSpot et Eloqua, qui utilisent elles-mêmes des cookies pour suivre les spectateurs. Lors de l'utilisation des profils d'audience, un cookie Brightcove est utilisé.
    Brightcove Audience est-il activé pour tous les clients ?
    Non. Audience est disponible en tant que module complémentaire à l'achat de Video Marketing Suite (VMS). Les profils d'audience, cependant, sont disponibles gratuitement pour tous les clients de Brightcove. Vous pouvez le considérer comme « Audience-lite ».
    Qui est admissible aux profils d'audience.
    À partir du 4e trimestre 2017, chaque client VMS aura accès gratuitement aux profils d'audience. Les clients devront l'activer pour commencer le suivi des données. Les clients Enterprise Video Suite (EVS) auront également le suivi Audience SSO disponible.

    Suivi et limitations des visionneuses

    Quelles sont les limites des profils d'audience par rapport au module Audience complet ?
    • Aucune intégration avec des systèmes tiers
    • Aucun accès à l'API
    • Pas de formulaires de prospect personnalisés (formulaires de prospect Audience uniquement)
    • 1000 vues/période de capture maximale de 24 heures
    Vous pouvez toujours exporter manuellement les données d'audience vers un fichier CSV ou XLS.
    Quelles sont les limitations/exigences d'Audience lorsqu'elle est connectée à une plateforme d'automatisation marketing ?
    Les limites et les exigences sont décrites dans la documentation de chaque plateforme :
    • Oracle Eloqua
    • HubSpot (API côté client ou API REST)
    • Marketo (Munchkin ou API REST)
    • Salesforce
    Y a-t-il des limites quant à la fréquence d'interrogation de l'API Audience ? Je sais qu'il y a une limite de 1000 sur les résultats de recherche vidéo, mais existe-t-il d'autres majuscules liées à la fréquence à ce point de terminaison API ?
    Le nombre maximal d'événements de vue par appel d'API est en fait 100. Il n'y a pas de limite de taux en place, mais si vous essayez de l'appeler trop, cela ralentira probablement un peu.
    Existe-t-il un moyen d'obtenir les données d'audience plus rapidement que le délai de 3 heures actuellement présent ? J'aimerais obtenir des mesures « proches des temps réel ».
    Non. Si vous voulez des mesures en temps réel, vous devez écrire votre propre plugin et envoyer des données à votre propre système.
    Quel est le déclencheur d'une vue d'événement à envoyer à Audience ? Est-ce à la fin de la vidéo, lorsque la fenêtre du lecteur est fermée, ou autre chose ? Que se passe-t-il si l'utilisateur ne termine jamais la vidéo et ne ferme pas sa fenêtre de lecteur ?
    Cela varie, mais, fondamentalement, le plugin Audience envoie un battement de cœur au serveur toutes les 10 secondes.
    Quand Audience considère-t-il que le suivi sur une vue est « terminé » ? Est-ce lorsque la fenêtre du lecteur est fermée ou si elle complète la vidéo ? Que se passe-t-il s'ils regardent partway, mettent en pause la vidéo et ne ferment jamais la fenêtre du lecteur ?
    C'est quand 15 minutes se sont écoulées depuis la dernière fois que nous avons vu un rythme cardiaque, essentiellement le manque d'activité.
    Est-il possible que les données du spectateur changent une fois qu'elles sont disponibles dans les données de visualisation de l'audience ? Ces données d'événement seront-elles jamais mises à jour à l'avenir par une sorte de consolidation des journaux ?
    Une fois que les données sont disponibles dans Audience via l'interface utilisateur ou l'API, elles sont considérées comme « réglées », et elles ne changeront pas.

    Protection des données et conservation des dossiers

    Audience est-elle conforme au GDPR ?

    Oui, Audience permet aux clients d'agir au nom de leurs utilisateurs, en recherchant et en purgeant les données qu'ils ont collectées via Audience.

    https://audience.support.brightcove.com/admin/managing-viewer-data.html

    Combien de temps peut-on conserver les données dans Audience ?
    Les intégrations MAP peuvent être réglées sur un maximum de 6 mois, et les intégrations non-MAP peuvent être réglées sur 3 ans.
    Combien de temps les recherches GUID sont-elles mises en cache ?
    30 jours pour les données connues et 7 jours pour inconnu (s'applique uniquement aux intégrations MAP, aucune recherche requise pour les intégrations non-MAP)

    Automatisation marketing et analyse des visionneuses

    Qu'est-ce qui diffère du suivi vidéo de n'importe quel autre média dans ma plateforme d'automatisation marketing ?
    Contrairement à une page Web ou à un livre blanc téléchargeable, avec une vidéo, vous pouvez déterminer exactement la quantité de vidéo regardée par le spectateur et, par conséquent, exactement à quel contenu il a été exposé.
    Comment vous connecter à mon compte Eloqua/Marketo/Hubspot ?
    Vous pouvez vous connecter à votre compte Eloqua/Marketo/Hubspot directement via Video Marketing Suite dans le module Audience. À partir de là, vous pouvez créer des lecteurs et des formulaires de prospect avec le suivi MAP intégré, de sorte que vous pouvez commencer à suivre l'analyse vidéo immédiatement.
    Puis-je suivre les vues sur mes pages Eloqua/Marketo/Hubspot ainsi que sur mes pages de destination non-eloqua/marketo/Hubspot (sitecore/WordPresss/Gallery etc) ?
    Oui, tant que le lecteur utilisé est « Audience activé » (le plugin Audience est installé), toutes les vues seront suivies, que le joueur réside sur une page de destination MAP ou sur toute autre page de destination.
    Que puis-je suivre avec cette intégration ?
    Grâce à notre intégration, vous pouvez suivre les analyses de visualisation vidéo (vidéo, ID vidéo, pourcentage de visualisation, temps regardé) pour les contacts connus déjà dans votre base de données.
    Qu'en est-il des contacts inconnus ? Je peux les suivre aussi ?
    Avec Brightcove Audience, vous pouvez suivre les contacts inconnus, puis lorsqu'ils envoient un formulaire Eloqua/HubSpot (devenu connu), Audience peut associer rétroactivement les informations de son auditoire à ce nouveau contact. L'audience retransmet chaque jour les informations de visualisation afin de faire correspondre les données de vue des contacts inconnus à votre instance Oracle Eloqua/HubSpot.
    Où mes données de visualisation vidéo apparaîtraient-elles dans mon MAP ?
    Cela dépend du MAP que vous utilisez. Reportez-vous à la documentation suivante pour Oracle Eloqua, Marketo et HubSpot.
    Où dans Salesforce mes données de visualisation vidéo apparaîtraient-elles ?
    Les vues vidéo sont fournies dans Salesforce sous la forme d'une tâche sous le contact du spectateur dans la section Activité. Pour obtenir un exemple, reportez-vous à la section Affichage des données vidéo dans Salesforce .
    Puis-je être averti avec Salesforce lorsqu'un de mes contacts a regardé des vidéos ?
    Pas directement. Toutefois, vous pouvez exécuter des rapports qui vous avertiraient de l'activité d'un contact.
    Y a-t-il une différence entre le suivi avec Salesforce et Oracle Eloqua/Marketo ?
    La plus grande différence est que Salesforce n'utilise pas de cookies pour suivre les utilisateurs. Par conséquent, un identifiant unique doit être transmis pour identifier le spectateur. Voir ce document pour plus d'informations. En outre, Salesforce, en tant que CRM, n'offre pas de suivi automatisé ou de notation de prospect. Toutefois, vous pouvez créer des tâches dans Salesforce, qui sont ensuite affectées à votre équipe commerciale pour des suivis réels.
    Audience a une option d'intégration « personnalisée », à quelles autres plateformes Audience peut-elle intégrer en utilisant cette option ?

    Le public peut « s'intégrer » à n'importe quelle plate-forme de données qui :

    1. Utilise un identifiant unique pour chaque enregistrement de contact
    2. A une API qui peut recevoir des données supplémentaires à partir de plates-formes tierces (comme Audience)

    Formulaires de plomb du joueur

    Quand Audience affichera-t-elle un formulaire de prospect ?
    Le paramètre par défaut affiche uniquement le formulaire prospect pour un utilisateur inconnu dans votre base de données MAP (ceux que vous souhaitez convertir en utilisateur connu). Vous pouvez le remplacer dans les paramètres si vous choisissez de toujours afficher le formulaire prospect. Ceci est très utile pour les tests. Les utilisateurs connus sont automatiquement suivis et ces données sont associées au contact dans votre MAP sans qu'il soit nécessaire d'afficher un formulaire.
    Puis-je personnaliser les formulaires de prospecte ?
    Vous pouvez ajuster le texte et les champs affichés à l'aide de l'interface utilisateur. Vous ne pouvez pas le modifier au-delà de ce qui est disponible dans l'interface utilisateur (Gallery peut mettre à jour certains styles pour que les formulaires correspondent au site)

    Sujets généraux

    Quel code d'intégration dois-je utiliser ? Iframe ou en page ?
    Pour que Audience fonctionne, il est nécessaire d'utiliser le code intégré avancé. Si le code d'intégration Standard (iframe) est utilisé, le lecteur est isolé de la page, ce qui signifie que le plugin Audience ne peut pas voir l'identifiant unique, et donc tous les utilisateurs seront considérés comme « inconnus ».
    Qu'est-ce que l'automatisation marketing ?
    L'automatisation du marketing est une technologie qui suit l'interaction lead/utilisateur avec différents canaux de marketing en ligne (email, web, formulaires, contenu, vidéo, social, etc.). L'automatisation du marketing aide les marketeurs à commercialiser plus efficacement sur ces canaux et automatise les tâches marketing répétitives (telles que les communications par e-mail). L'automatisation du marketing permet au marketeur de configurer des tâches automatisées et personnalisées basées sur des variables prédéfinies. Par exemple, si un utilisateur regarde 75 % d'une vidéo spécifique, envoyez-lui un e-mail sur ce sujet spécifique, mais s'il ne regarde que 12 %, envoyez-lui quelque chose d'autre.
    Brightcove est-elle une entreprise d'automatisation marketing ?
    Non, Brightcove fournit une solution complète pour le marketing vidéo. Avec notre solution, vous pouvez mettre vos vidéos en ligne, organiser votre contenu, créer des lecteurs, créer des portails vidéo, suivre des analyses et intégrer votre propre plateforme d'automatisation marketing existante. Brightcove intègre l'analyse vidéo dans les plates-formes d'automatisation marketing.
    Comment puis-je vérifier si une activité personnalisée Marketo est correctement configurée ?
    Audience a une URL cachée qui offre des outils de débogage pour certains types d'intégrations, Marketo REST étant un. https://studio.brightcove.com/products/videocloud/audience/debug
    Que faut-il faire pour configurer une intégration personnalisée ?
    Trois choses doivent être faites :
    1. Vous devez exposer un identifiant unique dans l'URL ou le code de la page que l'audience peut capturer.
    2. Créez un script qui utilise les API Audience pour récupérer les données sous forme de fichier JSON (Brightcove Global Services peut vous aider).
    3. Mappez les données aux champs de votre base de données de destination (pensez à fusionner des feuilles de calcul).
    La documentation de support suivante couvre ces tâches :
    • Création d'intégrations personnalisées avec le module Audience - Couvre comment fournir un identificateur unique que l'audience peut capturer
    • API Audience - Couvre comment récupérer par programmation toutes les données sous forme de fichier JSON que vous pouvez ensuite importer dans votre système de traitement de données de choix
    Comment Audience pousse-t-elle les données CDO vers Eloqua ?
    Nous utilisons l'API de masse d'objets personnalisés. https://docs.oracle.com/cloud/latest/marketingcs_gs/OMCAC/api-bulk-2.0-customobjects.html
    Comment Audience pousse-t-elle l'activité des actifs externes ? Comment puis-je contrôler les campagnes, les ressources externes et les types d'activité utilisés ?
    Nous utilisons l'API d'activité externe (https://docs.oracle.com/cloud/latest/marketingcs_gs/OMCAC/op-api-rest-2.0-data-activity-post.html). Il n'y a aucun contrôle sur le nom de la campagne, le nom ou le type de ressource. Nous créons simplement une campagne par nous-mêmes (nommée BrightCoveVideoPlay), puis procédons à la création d'activités, également appelées BrightCoveVideoPlay.
    Quelles adresses IP puis-je mettre en liste blanche pour que Audience puisse parler au service X ?
    Audience a établi un ensemble d'adresses IP qui doivent être utilisées pour la liste blanche IP. Pour afficher la liste, modifiez une connexion Eloqua et cliquez sur Impossible de voir les données d'audience dans Eloqua ? pour voir la liste des adresses IP.
