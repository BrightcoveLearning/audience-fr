    Identification des visiteurs par le module Audience

    Dans cette rubrique, vous découvrirez les différentes façons dont le module Audience tente d'identifier les spectateurs.

    Le module Audience tente d'identifier les spectateurs à l'aide de diverses sources de données :

    • Cookie public
    • GUID Oracle Eloqua
    • Biscuit Marketo Munchkin
    • Paramètre d'URL Salesforce ID
    • Attribut HTML d'ID Salesforce
    • Cookie HubSpot UTK
    • Paramètre d'e-mail

    Le module Audience respecte les paramètres du navigateur de l'utilisateur concernant Do Not Track.

    Remarque : Lorsque vous utilisez une intégration personnalisée, vous êtes responsable de l'identification de la visionneuse. Pour plus d'informations, voir Création d'intégrations personnalisées avec le module Audience.

    Cookie public

    Les lecteurs compatibles avec le public définiront un cookie tiers et l'utiliseront pour associer les données de visualisation au même utilisateur si le suivi de l'audience est activé pour leur compte et que le lecteur les utilise. Si un utilisateur remplit un formulaire de prospect d'audience, Audience créera un profil pour cette visionneuse et toutes ses données de visualisation seront associées à ce profil.

    Le cookie défini par Audience est nommé _bc_uuid.

    GUID Oracle Eloqua

    Les joueurs compatibles avec le public avec une intégration Eloqua invoqueront les API d'Eloqua pour obtenir l'ID de visiteur, souvent appelé GUID Eloqua. Cet ID est capturé dans les données de visualisation du module Audience et est utilisé pour synchroniser les données de visualisation avec Eloqua.

    Eloqua utilise son propre ensemble de cookies et lorsqu'un lecteur Brightcove est configuré avec une intégration Eloqua, Audience capture le GUID dans le cookie Eloqua et communique avec Eloqua côté serveur, pour essayer de comprendre qui est l'utilisateur.

    Biscuit Marketo Munchkin

    Les joueurs compatibles avec le public avec une intégration Marketo se réfèrent au cookie Munchkin (_mkto_trk) créé dans le navigateur de l'utilisateur. Cette valeur est capturée dans les données d'affichage du module Audience et est utilisée pour synchroniser les données de visualisation avec Marketo.

    Paramètre d'URL Salesforce ID

    Pour les intégrations Salesforce, le plugin peut tenter d'identifier la visionneuse en vérifiant l'URL du paramètre appelé bcsfid. Cela permet aux utilisateurs de Salesforce d'envoyer des campagnes par e-mail avec une URL de page de destination générée de manière unique à des fins de suivi en fournissant l'ID Salesforce du prospect dans l'URL.

    Attribut HTML d'ID Salesforce

    Pour les intégrations Salesforce, si le plugin ne parvient pas à identifier la visionneuse en vérifiant le paramètre d'URL bcsfid , il recherche l'attribut data-bc-sfid sur l' <video> élément de l'embed HTML pour la vidéo et utiliser sa valeur comme ID Salesforce du prospect si elle est définie.

    Cookie HubSpot UTK

    Pour les intégrations HubSpot, le plugin injecte le code de suivi JavaScript de HubSpot qui laisse tomber un cookie hubspotutk sur le navigateur de l'utilisateur. Ce cookie est utilisé pour garder une trace de l'identité d'un visiteur dans HubSpot et associer les données de visualisation de Brightcove à un contact. Si l'utilisateur dispose déjà d'un cookie hubspotutk lorsque le lecteur charge, Audience fera une demande d'API à HubSpot et demande si la valeur du cookie correspond à un contact connu, et si oui, Audience associera les données de visualisation à cet utilisateur.

    Paramètre d'e-mail

    Le module Audience peut également être indiqué qui est l'utilisateur via une adresse e-mail. L'audience recherchera un paramètre d'URL intitulé bcemail. Ceci est utile pour les campagnes par e-mail avec une URL de page de destination qui a été générée de manière unique à des fins de suivi. Si vous envoyez un e-mail par lots à une liste, que ce soit à partir d'une plateforme marketing ou d'un service de messagerie tiers comme MailChimp, vous devez configurer le paramètre de messagerie dans les paramètres de la plate-forme d'envoi pour mettre à jour dynamiquement le lien sur lequel les utilisateurs cliquent. Audience utilisera ensuite ce paramètre de messagerie pour rechercher le côté serveur de l'utilisateur et synchroniser les données d'affichage. Le paramètre email peut également être incorporé avec le code d'intégration du lecteur.

    Exemple d'URL

    http://go.brightcove.com/be-video-hero-cust?bcemail=msmith@brightcove.com

    Si le plugin Audience ne trouve pas d'adresse e-mail via le paramètre d'URL bcemail , il vérifie alors l' <video> élément du code de publication HTML incorporé. Il recherchera l'attribut data-bc-email sur l'élément vidéo et utilisera sa valeur comme adresse e-mail si elle est définie.

    Exemple d'intégration

    <vidéo
id="joueur »
data-account="0000000001"
data-player="par défaut »
data-video-id="0000000002"
   data-bc-email= » msmith@brightcove.com »
classe="video-js » contrôles >
</vidéo >

    Ne pas suivre

    Les lecteurs compatibles avec le public respectent les paramètres du navigateur d'un spectateur concernant Do Not Track (DNT) et n'utiliseront pas ou ne définiront pas son cookie normal s'il voit que Do Not Track est activé. Le plugin du lecteur peut également être invité à ne jamais suivre si un client Video Cloud souhaite l'éteindre pour tous ses spectateurs ou sur une base utilisateur par utilisateur. Ceci est fait en utilisant le code d'intégration dans la page et en définissant data-do-not-track la valeur sur une valeur de 1.

    <video-js data-video-id="9240259371001"
data-account="3491906377"
data-do-not-track="1"
data-player="7dccc39d-0e2d-4b27-bea9-af44b2166148"
data-embed="default » class="video-js » contrôles > < /video-js >
<script src= » https://players.brightcove.net/1486906377/7dccc39d-0e2d-4b27-bea9-af44b2166148_default/index.min.js "> < /script >

    Oracle Eloqua

    Eloqua a déclaré qu'ils ne traitent pas (ignorent fondamentalement) ne suivent pas les requêtes et retourneront un GUID Eloqua quel que soit. Actuellement, Audience ne gère pas cela de manière spéciale, c'est-à-dire ne pas utiliser l'API JavaScript Eloqua lorsque Audience voit DNT. Eloqua est également favorable à un mode strict qui satisfait aux exigences de l'UE en matière de confidentialité. Lorsque cette option est activée pour un compte Eloqua, les utilisateurs finaux ne sont pas suivis à moins qu'ils ne s'y inscrivent via un formulaire. Les cookies supprimés dans les deux cas sont documentés dans le document Oracle Quels cookies sont supprimés lorsque vous accédez à une page suivie d'Eloqua.

    Marketo

    Marketo prend en charge le DNT mais laisse au client le soin de l'activer/désactiver pour son compte particulier. Pour plus d'informations, consultez le document Marketo Modifier les paramètres de support du navigateur « Ne pas suivre ».

    HubSpot

    HubSpot respecte le DNT selon ce document.
