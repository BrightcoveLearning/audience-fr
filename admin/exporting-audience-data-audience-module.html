    Exportation de données d'audience à partir du module Audience

    Dans cette rubrique, vous apprendrez comment exporter les données d'événement de vue et les données de formulaire de prospect.

    Les événements de vue sont enregistrés au fur et à mesure que les vidéos sont regardées à l'aide de lecteurs compatibles Audience. Les données de prospect sont capturées lorsque les utilisateurs entrent des données dans les formulaires de prospect Audience Le module Audience propose deux types de formulaires de prospect :

    • Formulaires de prospect d'audience - Ces formulaires sont créés en sélectionnant parmi un ensemble de champs prédéfinis dans le module Audience. Ces formulaires enregistrent les données de prospect dans Video Cloud. Pour plus d'informations sur la création de formulaires de prospect d'audience, voir Création d'un formulaire de prospect d'audience.
    • Formulaires de prospect personnalisés - Ces formulaires sont créés à l'aide de votre plateforme d'automatisation marketing. Le code d'intégration du formulaire est ensuite utilisé dans le module Audience pour créer un formulaire de prospect personnalisé. Ces formulaires envoient des données de prospect à la plateforme d'automatisation du marketing. Pour plus d'informations sur la création de formulaires de prospect, consultez le document correspondant à votre plateforme marketing :
      • Eloqua
      • Marketo
      • Salesforce
      • HubSpot

    Lorsque vous exportez des données de prospect à partir du module Audience, seules les données capturées à l'aide des formulaires de prospect Audience Les données de prospect provenant de formulaires de prospect personnalisés seront stockées dans votre logiciel d'automatisation marketing.

    Pour exporter des données, procédez comme suit :

    1. Ouvrez le module Audience.
    2. Cliquez sur Exporter les données d'audience dans la navigation de gauche.
      Remarque : Vous devez être administrateur de compte pour accéder au menu Exporter les données.
    3. Sélectionnez une connexion de données à partir de partir de. La liste déroulante Connexion de données répertorie toutes les intégrations configurées.
    4. Sélectionnez les données à exporter.
      • Afficher les événements - Enregistré par les lecteurs compatibles avec le public
      • Profils - Capturés à l'aide de formulaires
    5. Utilisez le sélecteur de dates pour sélectionner une date de début et une date de fin.
    6. Sélectionnez un Format d'exportation.
      • Valeurs séparées par des virgules (CSV)
      • Excel (XLS)
    7. Cliquez sur Exporter pour télécharger les données de prospect.
    8. Les données du formulaire de prospect et de l'événement de vue seront téléchargées dans des fichiers distincts. Les données despistes contiennent les données saisies dans le formulaire de prospect ainsi que les champs suivants :
      • ExternalId - Il s'agit de l'ID externe qui apparaît sur la page Toutes les activités récentes. Dans le cas d'Oracle Eloqua, c'est le guid Eloqua. Avec Marketo, c'est le guid Marketo
      • PlayerID - ID du joueur utilisé lors de l'affichage du formulaire
      • VideoID - ID vidéo de la vidéo en cours de lecture lors de l'affichage du formulaire
      • PageURL - URL de la page affichant le formulaire de prospect (expériences dans la galerie uniquement)
      • CreateDAT - Date et heure de saisie des données
      • expérience - Expérience de la galerie qui a recueilli les données (expériences dans la galerie uniquement)

      Les donnéesd'événements d'affichage contiendront les champs suivants :

      • ExternalId - Il s'agit de l'ID externe qui apparaît sur la page Toutes les activités récentes. Dans le cas d'Oracle Eloqua, c'est le guid Eloqua. Avec Marketo, c'est le guid Marketo
      • VideoID - ID vidéo de la vidéo
      • VideonName - Nom de la vidéo
      • regardé - Pourcentage de vidéos regardées (au pourcentage le plus proche)
      • TimeWatched - Temps total regardé
      • PlayerID - ID de joueur du lecteur utilisé pour regarder la vidéo
      • PageURL - URL de la page contenant le lecteur
      • CreateDAT - Date et heure de saisie des données
      • UpdateDAT - Date et heure de la dernière sauvegarde de l'événement, ce qui se traduit approximativement par le moment où le spectateur a cessé de regarder ; si la vidéo ne dure qu'une minute, CreateDAT et UpdateDAT seront très proches
      • isSynced - Indique si cet événement de vue a été automatiquement exporté vers l'outil d'automatisation marketing de l'éditeur
      • TrackIngID - ID de suivi passé en tant que paramètre d'URL. Pour plus d'informations sur l'utilisation d'un ID de suivi, reportez-vous à la section Création d'intégrations personnalisées avec le module Audience.
      • Champs UTM - UtmSource, utmMedium, UtmCampaign, UtmTerm et UTMContent. Pour plus d'informations sur l'utilisation des champs UTM, voir Utilisation des champs UTM dans le module Audience.
      • metric1-3, event1-3- Utilisé lors de l'utilisation de mesures et d'événements personnalisés
